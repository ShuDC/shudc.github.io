可转债套利计算器

Deven

发布于：2022年9月1日
次浏览

一、需求及方案

一.套利原理：
   可转债价值为100，但转为股票后的价值为101。其有溢价差，因此可进行套利。

二.套利思路（风险对冲套:）
  方法：在某个溢价率高的时刻，买入可转债并转股，同时对该股票融券（融多少券？融可转债转股后的价值）。即买入100的元的可转债，卖出101元的证券。到手1元。次日：可转债换成股票，直接点击还券即可。无需交易。
  风险：风险极小。在第一天就已经锁定了收益，第二天只是转股证券去还融入证券，即左手倒右手。。
  交易成本：最低交易费用为万11.6。融券利息为万2。也即是0.13.6%。因为有零数和整数的差别，假设为0.15%。

三、 套利买卖计算
  计算融多少券 ？买多少可转债？
  融券单位：1手100股。每手价格大多为几百到几千。
  转债单位：1手10张。每手价格多为1千到几千。

  设可用资金为10W，分两批买入。每次买入价格约为5W。
  设溢价差为c%。
  设可转债买入价格为a，买入可转债的价格为10a。设买x 手转股后价格为m=10a（1+c%）x。
  设股票卖出价格为b，融券卖出股票的价格为，设卖y 手，则总卖价n=100b *y。

  目的是让转股后的价值10a*（1+m%）与100y 近乎相等即可。表现为买入可转债的价格略低于融券价格 。
  如果m>n，则意味着多买了转债，转股后会承担明天价格变化。多买的这部分明天涨就赚，跌就亏。
  如果n>m，则意味着融券多卖了股票，转股后会承担明天价格变化。融券多买的这部分明天涨就亏，跌就赚。
  所以两者之间差距越小越好。

  引入计算公式，实现量化交易。根据公式 

m=10a*（1+m%）*x。  
n=100b *y   
求取x,y的合理值。可先算的x/y 的比例。  
x/y=100b/(10a(1+c))

举例：
a=109.6 c=1.71   转股后单价111.48
b=4.37 .
则x/y =0.392/1     可得x 买入0.392手，y 需要买入1手。近取整得x买392手，y买1000手。 
y/x =100/39.2=2.55。也可以有x买100手，y买255手。取整得x买20手，y买51手。。
最佳买法有： x买20手，y买51手。

因此我们需要设计一个用来计算出买卖股票和可转债数量的计算器。
该计算器不考虑跨平台，初步定为windows桌面窗体应用程序，即.exe程序。 非控制台应用程序。

二、设计步骤

一、参考资料
  microsoft官方参考文档： Windows桌面应用开发

二、开发方案
  用什么工具和语言能开发windows桌面程序？一般有以下三大方案。
  1.采用.net平台/框架开发，其主流语言为C#。
  2.采用java平台开发，其主流语言为java。
  3.采用QT平台开发，其主流语言为C++。
  我们选择第一大方案，采用.net平台，使用语言为C#，使用IDE为Visual Studio。

三、.net平台认知
  .NET生态系统主要包括三大高级组件：.NET Framework,.NET Core, Xamarin。
  .NET Framework：支持windows和web应用程序。
  .NET Core：是一种新的开源和跨平台框架,用于为包括 Windows,Mac和 Linux在內的所有操作系统构建应用程序。
  Xamarin:无庸置疑,当您想使用C#构建移动(OS, Android和windows Mobile)应用程序时, Xamarin是您唯一的选择。
  这里我们优先选择.NET Framework框架，原因是因为他常用，而且我们也不需要跨平台。

四、.net下的常用应用类型
  1.控制台应用程序：这种程序的执行就好像在一个DOS窗口中执行一样，没有自己的界面。
  2.winform窗体应用程序：Windows 窗体是用于托管型 Windows 应用程序的原始平台，具有一个轻型 UI 模型和对 .NET 或 .NET Framework 的访问权限。 它擅长帮助开发人员快速开始构建应用程序，即使对于刚接触该平台的开发人员也是如此。 这是一种基于窗体的快速应用程序开发平台，其中包含大量内置的可视化和非可视化拖放控件。 Windows 窗体不使用 XAML。
  3.WPF应用程序：WPF 是用于托管型 Windows 应用程序的功能完善的平台，具有对 .NET 或 .NET Framework 的访问权限，并使用 XAML 标记将 UI 与代码分隔开来。 此平台旨在用于需要复杂 UI、自定义样式和图形密集型方案的桌面应用程序。 WPF 开发技能类似于 WinUI 3 开发技能，因此从 WPF 迁移到 WinUI 3 比从 Windows 窗体迁移更容易。尽管 .NET 是一种跨平台技术，但 WPF 并不是，它仅在 Windows 上运行。大多数的WPF程序可能同时包含程序代码和 XAML。
  winform和wpf的区别？
  WPF比WinForm先进是不容置疑的。winform的控件更多，但是wpf的界面可以更炫酷，更重要的是wpf对于屏幕的自适应比winform优秀得多。
  这里我们采用winform类型的应用程序，因为我们不需要多么炫酷的界面，简单好用才是王道。

//第一部分：标准的头文件，引用名称为System.***的命名空间。
using System;
using System.Collections.Generic;
using System.ComponentModel;
using System.Data;
using System.Drawing;
using System.Linq;
using System.Text;
using System.Threading.Tasks;
using System.Windows.Forms;


//第二部分：核心代码
namespace 交易量计算器             
{
  public partial class Form1 : Form     
  {

    //第2.1节  界面部分
    public Form1()              
    {
        InitializeComponent();
        Text = "可转债套利计算器";   //这个类的标题属性定义为交易量计算器

        this.listView1.GridLines = true; //显示表格线
        this.listView1.View = View.Details;//显示表格细节
        this.listView1.LabelEdit = true; //是否可编辑,ListView只可编辑第一列。
        this.listView1.Scrollable = true;//有滚动条
        this.listView1.HeaderStyle = ColumnHeaderStyle.Clickable;//对表头进行设置
        this.listView1.FullRowSelect = true;//是否可以选择行

        //this.listView1.HotTracking = true;// 当选择此属性时则HoverSelection自动为true和Activation属性为oneClick
        //this.listView1.HoverSelection = true;
        //this.listView1.Activation = ItemActivation.Standard; //
        //添加表头
        listView1.Columns.Add("", 0, HorizontalAlignment.Center);
        listView1.Columns.Add("序号", 50, HorizontalAlignment.Center);
        listView1.Columns.Add("连分数", 80, HorizontalAlignment.Center);
        listView1.Columns.Add("买入可转债（手）", 150, HorizontalAlignment.Center);
        listView1.Columns.Add("融券卖出（手）", 150, HorizontalAlignment.Center);
        listView1.Columns.Add("比例", 100, HorizontalAlignment.Center);            
    }

    //第2.2节 功能以及算法部分
    private void jisuan_button_Click(object sender, EventArgs e)
    {

        //01:清空文本框 以及删除ListView中的内容
        foreach (Control ctrl in Controls)
        {
            if (ctrl is TextBox)
                ctrl.Text = "";
        }           
        listView1.Items.Clear();    

        //02:读取数字输入框的值，将其乘以100，并转换成int数据类型，同时定义每手转债的标准面值1000；
        double num = Convert.ToDouble(numinput.Value);        
        double num100 = num * 100;
        int x = Convert.ToInt32(num100);      //方式2：int i = (int)a;                 
        int y = 1000;

        //03:求取每手转股价标准面值和可转债标准面值比例的第一个连分数；并在文本框2中显示出来
        int[] a = new int[100];
        double r = x/y;                     
        double shang = Math.Floor(r);       
        a[0] = Convert.ToInt32(shang);
        int y1 = x % y;                     
        x = y;
        y = y1;
        textBox2.Text = a[0].ToString() + ";";

        //04:添加第一个连分数对应的数据到 ListView的第一项
        ListViewItem lvItem0 = new ListViewItem();
        lvItem0.SubItems.Clear();
        lvItem0.SubItems[0].Text = "第一行";
        lvItem0.SubItems.Add("A0");
        lvItem0.SubItems.Add(Convert.ToString(a[0]));
        lvItem0.SubItems.Add(Convert.ToString(a[0]));
        lvItem0.SubItems.Add("1");
        lvItem0.SubItems.Add("*.*****");        
        listView1.Items.Add(lvItem0);


        //05:如果求取一个连分数后的余数不为0，则进行下一步；
        if (y != 0)
        {        
            double shang1;
            int  m;
            int j;
            int i = 0;
            double[] z = new double[100];
            double[] q = new double[100];
            double[] c = new double[100];
            double[] v = new double[100];
            double[] w = new double[100];
            double bl;
            do
            {
                //循环求取连分数后续项。直至余数为0；并将后续连分数在文本框2中显示出来;
                r = x / y;              
                shang1 = Math.Floor(r); 
                int n = Convert.ToInt32(shang1);
                m =x % y;           
                x = y;                                 
                y = m;
                i = i + 1;
                a[i] = n;
                textBox2.Text = textBox2.Text + a[i] + ",";

                //根据连分数求得相近的分子和分母。并将分子v[1]赋值给可转债，分母w[1]赋值给股票
                j = i;
                w[j] = a[j];
                v[j] = a[j - 1] * w[j] + 1;
                while (j != 1)     
                {
                    j = j - 1;
                    w[j] = v[j + 1];
                    v[j] = a[j - 1] * w[j] + w[j + 1];                       
                } 
                z[i] = v[j];
                q[i] = w[j];
                bl = z[i] / q[i];
                c[i]= Math.Round(bl, 5);

                //循环多少次就输出多少数据
                ListViewItem lvi = new ListViewItem();
                lvi.ImageIndex = i;
                lvi.Text = "subitem" + i;
                lvi.SubItems.Add("A" + Convert.ToString(i));    //序号这一列
                lvi.SubItems.Add(Convert.ToString(a[i]));       //连分数这一列
                lvi.SubItems.Add(Convert.ToString(z[i]));            //可转债数这一列
                lvi.SubItems.Add(Convert.ToString(q[i]));            //融券数这一列
                lvi.SubItems.Add(Convert.ToString(c[i]));            // 差值这一列
                listView1.Items.Add(lvi);
            } while(y != 0);
        }
        //05:结束

    }
    //第2.2节结束

  }
}
//第二部分结束
